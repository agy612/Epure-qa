# epure QA 작성 가이드

## 시작 방법
새 날짜로 QA 시작할 때:
> "오늘 날짜로 QA 파일 새로 만들어줘"

이후 이슈를 말하면 바로 CSV에 추가해줌.

---

## CSV 파일 규칙

- 파일명: `epure_qa_YYMMDD.csv` (예: `epure_qa_260226.csv`)
- 인코딩: **utf-8-sig (BOM 포함)**
- 위치: 프로젝트 루트

### 컬럼 구성
```
중요도, 구분, 유형, 페이지, 이슈타입, 이슈내용
```

| 컬럼 | 설명 | 값 |
|------|------|-----|
| 중요도 | 우선순위 | **비워둠** (직접 작성) |
| 구분 | 플랫폼 | 매니저웹 / 클라이언트웹 / 방송송출앱 / 서버 |
| 유형 | 사용자 유형 | 셀러 / 클라이언트 |
| 페이지 | 해당 페이지명 | 홈 / 라이브 예약 / 마이페이지 등 |
| 이슈타입 | 이슈 성격 | 기능 에러 / UI 개선 / 기능 개선 |
| 이슈내용 | 이슈 설명 | 자유 기술 |

---

## 자동 분류 규칙

### 구분 → 유형 자동 매핑
| 구분 | 유형 |
|------|------|
| 매니저웹 | 셀러 |
| 클라이언트웹 | 클라이언트 |
| 방송송출앱 | 셀러 |

### 이슈타입 판단 기준
| 이슈타입 | 해당 상황 |
|---------|---------|
| 기능 에러 | 동작이 아예 안 되거나 잘못 동작하는 경우 |
| UI 개선 | 화면 디자인/레이아웃 개선이 필요한 경우 |
| 기능 개선 | 기능은 있으나 동작 방식 개선이 필요하거나 새 기능 추가가 필요한 경우 |

---

## 정렬 규칙
- 피그마 참고 항목은 하단에 모음
- 구분 순서: 매니저웹 → 방송송출앱 → 클라이언트웹

정렬 요청 시:
> "피그마 참고 있는 것들 아래로 모아주고 구분별로 정렬해줘"

---

## CSV 작성 규칙
- 이슈내용에 쉼표 포함 시 큰따옴표로 감싸기
- 큰따옴표가 내용에 포함 시 `""` 로 이스케이프
- "피그마 참고" 항목은 이슈내용 뒤에 `(피그마 참고)` 추가

---

## 이슈 입력 예시

> "매니저웹 라이브 예약 페이지에서 썸네일 이미지 업로드 시 크기 제한 안내 없음"

→ `,매니저웹,셀러,라이브 예약,UI 개선,썸네일 이미지 업로드 시 파일 크기 제한 안내 없음`
